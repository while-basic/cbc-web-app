# Functions Quick Reference

## 10 Functions for Fade-Into-Background System

### Core Functions (Use Frequently)

1. **`surface_project`** - Project card ONLY when it's the focal point of breakthrough/inquiry
2. **`surface_question`** - Underlying questions/inquiry (redirects attention inward)
3. **`surface_connection`** - Cross-domain connections and systems thinking patterns
4. **`surface_insight`** - Key insights, mental artifacts, coined terms
5. **`surface_philosophy`** - Methodology and approach (how, not what)

### Supporting Functions (Use Strategically)

6. **`surface_coherence`** - Patterns that keep returning (center of gravity)
7. **`leave_space`** - Intentional silence when user is engaged
8. **`surface_media`** - Media ONLY when it's the subject of inquiry (not content)
9. **`track_conversation_depth`** - Internal tracking for fade mechanism
10. **`redirect_attention_inward`** - Force redirect from external to internal

---

## Usage by Conversation Stage

### Orientation (Early)
- `surface_project` (establish context)
- `surface_philosophy` (establish approach)
- `track_conversation_depth` (monitor)

### Deepening (Middle)
- `surface_question` (shift to inquiry)
- `surface_connection` (show patterns)
- `surface_coherence` (show structure)

### Established (Deep)
- `surface_insight` (depth)
- `leave_space` (withdrawal)
- `redirect_attention_inward` (as needed)

---

## Key Principles

✅ **DO:**
- Use functions to surface thinking, not outputs
- Focus on questions over answers
- Show patterns, not artifacts
- Leave space when appropriate

❌ **DON'T:**
- Use for promotion or showcasing
- Create skimmable content
- Optimize for discoverability
- Hold attention unnecessarily
